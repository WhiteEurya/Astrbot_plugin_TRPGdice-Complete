# TRPGdice-Complete TODO

本文档记录当前插件在三个核心方向上的功能现状和后续开发计划：CoC 功能、Log 管理、角色卡管理。不整理 DND 专项、牌堆、自定义回复、机器人管理等非核心骰子插件功能。

最后更新：2026-05-10

## 当前结构概览

- 入口与 AstrBot 命令注册主要在 `main.py`。
- 掷骰解析在 `component/roll/parser.py`。
- 掷骰与 CoC 判定核心在 `component/roll/dice.py`。
- CoC 规则、检定扩展入口、SAN 相关逻辑在 `component/coc/`。
- 角色卡存储、展示、名片、模板在 `component/pc/`。
- 日志存储和统计在 `component/logs/`。
- 先攻管理在 `component/combat/`。
- 输出文案在 `component/config.yaml`。

## CoC 功能

### 已实现

- `.ra` 普通技能/属性检定。
- `.rab` / `.rap` 奖励骰、惩罚骰检定。
- `.ra10#侦查70` 这类多轮检定。
- 从当前绑定角色卡读取技能值。
- @ 他人进行代骰检定。
- `.rc` 规则书检定入口。
- `.rah` / `.rch` 暗中技能检定入口，结果私聊发送。
- `.rav` / `.rcv` 对抗检定入口，支持 `技能名a点数/b点数`、`技能名/b点数` 和 `技能名 @对抗者`。
- 指定难度检定，例如 `.ra困难侦查70`、`.ra极难侦查70`。
- 技能修正值，例如 `.ra侦查+10`、`.ra侦查70+10`。
- `.sc` 理智检定，并自动扣除角色卡 SAN。
- SAN 大额损失时给出 `.ti` 提示。
- `.ti` 临时疯狂症状。
- `.li` 长期/不定性疯狂症状。
- `.en` 技能成长检定，并可回写角色卡。
- `.setcoc` 切换大成功/大失败规则。
- `.coc` / `.cocN` 随机生成 CoC 调查员基础属性。
- `.fire` 手枪三连发检定。

### 待实现或待完善

- `.rc` / `.rch` / `.rcv` 目前主要是命令入口区分，尚未真正和 `.ra` 房规检定做语义分离；需要明确“规则书检定”和“房规检定”的差异。
- `.rav` / `.rcv` 对抗检定目前是基础版：支持双方显式数值和 @ 对抗目标，但还缺少复杂平局规则细节配置。
- 暗中检定目前把结果发给发起者；还需要支持发送给 KP/指定用户。
- `.r 侦查+10` 这类普通掷骰中引用角色卡变量的能力尚未实现。
- `.setcoc details` 尚未实现，当前只能切换/查看基础规则。
- CoC 角色卡模板目前只有 `.st 模板` 文本模板，还没有一键录卡解析和兼容导入。
- SAN 损失后的临时疯狂流程仍是提示式，尚未自动串联 `.ti` 或记录疯狂状态。
- Delta Green 等 CoC 派生规则尚未评估和实现。
- 检定结果目前主要依赖文本输出，尚未形成结构化判定对象；后续日志统计、对抗检定、自动成长会受限。

## Log 管理

### 已实现

- `.log new [名称]` 新建日志会话。
- `.log off` 暂停记录。
- `.log on [名称]` 恢复记录。
- `.log end` 结束并导出 log-painter JSON。
- `.log list` 列出日志。
- `.log del <名称>` 删除日志。
- `.log get <名称>` 重新导出指定日志 JSON。
- `.log stat [名称] [--all]` 查看日志基础统计。
- `.log export <名称>` 导出纯文本日志。
- `.log ob add/del/list/clear` 管理当前日志 OB 名单。
- `.ob` 快捷切换自己在当前日志中的 OB 模式。
- 自动记录群聊消息。
- 自动记录骰点结果。
- 记录文本、用户 ID、昵称、时间戳、图片 URL。
- OB 用户消息会在日志中标记 `isObserver`，导出 JSON 时附带 `role: OB`。
- 导出为 log-painter 可读取的 JSON。
- log-painter 前端支持预览、染色、部分内容过滤、导出文本/HTML/Word 相关格式。

### 待实现或待完善

- `.log stat` 目前只有基础统计：消息数、参与者数、骰点数、图片数。尚未统计成功/失败次数、属性变化、SAN 变化、发言排行等。
- `.log export` 目前只返回导出文件名，尚未直接发送文件或生成下载链接。
- 缺少邮件发送日志功能。
- 缺少按群号代取日志能力，用于已解散或机器人已退出的群。
- 缺少日志权限控制，例如仅 KP/管理员可导出、删除、查看指定日志。
- OB/旁观者已有基础标记机制，但 log-painter 中仍缺少专门的“一键隐藏 OB 发言”过滤开关。
- 缺少日志管理后台或 UI，用于跨群检索、删除、导出所有群日志。
- log-painter 多格式导入兼容仍需完善，部分 importer 仍需要实际样本测试。
- 缺少上传到公共染色器或生成永久分享链接的服务端流程。
- 日志存储中仍存在测试群特殊分支，需要清理成配置化测试数据或移除。

## 角色卡管理

### 已实现

- `.pc new <角色名>` 创建角色卡。
- `.pc list` 列出当前群自己的角色卡。
- `.pc tag <名称|序号>` 绑定/切换当前角色卡。
- `.pc tag` 不带参数可解除绑定。
- `.pc rename` 重命名当前或指定角色卡。
- `.pc delete <名称|序号>` 删除角色卡。
- `.pc show` 查看角色卡属性。
- `.pc push` / `.pc fetch` / `.pc pull` 手动跨群同步角色卡。
- `.st 属性值` 设置属性。
- `.st 属性+值` / `.st 属性-值` / `.st 属性*值` 修改属性。
- `.st 属性骰子表达式` 用骰点结果修改属性。
- `.st 属性值 属性值...` 批量录入。
- `.st show` 查看当前角色卡属性。
- `.st show @某人` 查看他人当前绑定角色卡。
- `.st show <数字>` 查看高于阈值的属性。
- `.st del <属性>` 删除非核心属性。
- `.st clr` 清理非核心属性。
- `.st export` 导出录入表达式。
- `.st 模板` / `.st template` 输出 CoC 录卡模板。
- 属性同义词同步，例如 `str/力量`、`dex/敏捷`、`san/理智`。
- 自动维护部分派生属性，例如 `max_hp`、`max_san`。
- `.sn` 根据角色卡更新群名片。
- `.sn coc`、`.sn cocL`、`.sn none`、`.sn off` 基础名片模式。

### 待实现或待完善

- `.nn <角色名>` 快速设置当前角色名或昵称尚未实现。
- `.pc save` / `.pc load` 或更标准的跨群角色库流程尚未实现。
- 跨群同步目前仍偏手动，需要减少 `.pc push/fetch/pull` 操作，例如绑定后自动检测新版本。
- `.st 模板` 只是输出模板，尚未支持一键按模板录入。
- 尚未兼容 `.st <角色名>-属性值...` 这类简化卡格式。
- `.st 属性=值`、`.st 属性:值` 等更宽松录入格式尚未实现。
- `.st @某人 属性值` 代改他人角色卡尚未实现；目前主要支持查看和检定代骰。
- 保存计算表达式尚未实现，例如 `.st &手枪伤害=1d6+1`。
- 角色卡导入/导出兼容不足，尚未支持海豹、Dice!、纯文本卡、JSON 卡等常见格式。
- `.sn expr` 自定义表达式名片模式尚未实现。
- `.sn off` 目前只是本次命令生成空名片，尚未持久关闭自动名片更新。
- 缺少角色卡权限、锁定、备份、恢复机制。
- 缺少角色卡 Web 管理界面。
- `component/pc/store.py` 中仍有较多历史乱码注释和内联同义词表，后续应继续拆分为 `sync.py` / `schema.py` / `importer.py` 等模块。

## 当前优先级建议

1. 修正 CoC 检定语义：明确 `.ra` 与 `.rc` 的规则差异，并让 `.rc/.rch/.rcv` 真正使用规则书逻辑。
2. 完善 `.rav/.rcv`：补充平局细则配置和输出结构化。
3. 实现 `.r 侦查+10`：普通掷骰表达式可引用角色卡变量。
4. 完善 `.st` 录入：支持 `=`、`:`、模板一键录入、简化卡格式。
5. 完善 `.sn`：实现 `expr` 自定义模式和 `off` 持久关闭。
6. 改进跨群角色卡同步：自动检测、冲突提示、备份恢复。
7. 扩展 `.log stat`：统计检定成功/失败、SAN 变化、属性变化、发言排行。
8. 完善日志导出生态：文件发送、下载链接、权限控制、OB 过滤 UI、log-painter 多格式导入。
9. 清理历史乱码注释和硬编码测试分支，继续提升维护性。
